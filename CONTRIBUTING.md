# Разработка

Для обеспечения единообразия окружения при компиляции документации рекомендуется использовать **Docker**. Это позволяет избежать проблем с отсутствующими шрифтами или разными версиями пакетов TeX.

### Сборка через Docker

Используйте следующую команду для запуска сборки внутри контейнера:

```bash
docker run --rm -v .:/data ghcr.io/itmo-math-statistics-ls-2026/1-xetex-compiler:dev make pdf
```

**Примечание:** При запуске через Docker текущая директория монтируется в `/data`, что позволяет сохранять результат компиляции на вашей локальной машине.

> **Выбор тега образа:**
> * `:latest` — стабильная версия из ветки `main`.
> * `:dev` — актуальная версия для разработки из ветки `dev`.
> * `:pr-<номер>` — временные образы для проверки изменений в Pull Request (если был изменен Dockerfile).

### Команды Makefile

В проекте используется `Makefile` для автоматизации рутинных задач. Основные цели:

* **clean**: удаляет временные файлы компиляции и готовые PDF-файлы
* **pdf**: выполняет двойную компиляцию исходного кода через `xelatex` для корректного отображения ссылок и оглавления
